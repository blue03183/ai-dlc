# 테이블오더 서비스 요구사항 문서

## Intent Analysis

| 항목 | 내용 |
|---|---|
| **사용자 요청** | 테이블오더 서비스 신규 구축 |
| **요청 유형** | New Project (Greenfield) |
| **범위** | System-wide — 고객용 UI, 관리자 UI, 백엔드 서버, 데이터베이스 |
| **복잡도** | Complex — 멀티테넌트, 실시간 통신(SSE), 인증/세션 관리, 파일 업로드 |

## 1. 기술 스택 결정

| 영역 | 기술 |
|---|---|
| **프론트엔드 (고객용)** | React + TypeScript (별도 앱) |
| **프론트엔드 (관리자용)** | React + TypeScript (별도 앱) |
| **백엔드** | Node.js + NestJS + TypeScript |
| **데이터베이스** | MySQL |
| **실시간 통신** | Server-Sent Events (SSE) |
| **인증** | JWT 토큰 기반 |
| **배포 환경** | 로컬 개발 환경 우선 (향후 확장 고려) |

---

## 2. 아키텍처 결정

- **멀티테넌트**: 다중 매장 지원 — 각 매장이 독립적으로 운영
- **프론트엔드 분리**: 고객용 앱과 관리자용 앱을 별도 웹 애플리케이션으로 구성
- **이미지 관리**: 서버에 이미지 파일 업로드 기능 구현
- **관리자 계정**: 초기 관리자는 DB 시딩, 추가 계정은 기존 관리자가 생성
- **실시간 업데이트**: 관리자 화면 + 고객 화면 모두 SSE 기반 실시간 업데이트
- **규모 설계**: 대규모 (매장당 테이블 50개 이상, 관리자 5명 이상) 고려

---

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 고객용 기능

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 초기 설정 1회 수행 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 16시간 세션 유지

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본(홈) 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공 시: 주문 번호 표시, 장바구니 자동 비우기, 메뉴 화면으로 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- SSE 기반 주문 상태 실시간 업데이트

### 3.2 관리자용 기능

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 주문 목록 실시간 업데이트 (2초 이내)
- 테이블별 카드 형태 그리드/대시보드 레이아웃
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

#### FR-A03: 테이블 관리
- **테이블 초기 설정**: 테이블 번호/비밀번호 설정, 세션 생성
- **주문 삭제**: 확인 팝업 후 즉시 삭제, 총 주문액 재계산
- **테이블 세션 처리**: 이용 완료 시 주문 내역을 과거 이력으로 이동, 현재 주문/총액 리셋
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (등록/조회/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 (파일 업로드)
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

#### FR-A05: 관리자 계정 관리
- 초기 관리자 계정 DB 시딩
- 기존 관리자가 추가 관리자 계정 생성

## 4. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- SSE 기반 실시간 주문 표시 2초 이내
- 대규모 동시 접속 지원 (매장당 테이블 50개 이상)

### NFR-02: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적인 카드 기반 레이아웃
- 명확한 시각적 계층 구조

### NFR-03: 데이터 무결성
- 장바구니 로컬 저장 (새로고침 시 유지)
- 주문 이력 데이터베이스 영구 저장
- 세션 ID 기반 주문 그룹화

### NFR-04: 테스트
- 단위 테스트 + 통합 테스트 포함

## 5. 제외 기능 (Out of Scope)

requirements/constraints.md에 정의된 대로:
- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS, 2FA)
- 이미지 리사이징/최적화, CMS, 광고

- 고급 기능 (분석 대시보드, 매출 리포트, 재고, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도, 번역)
