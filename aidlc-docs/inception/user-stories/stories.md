# 테이블오더 서비스 — 사용자 스토리

---

## Feature 1: 인증 및 세션 관리

### US-101: 테이블 태블릿 자동 로그인
**As a** 고객 (P-C01, P-C02, P-C03)
**I want to** 태블릿을 켜면 자동으로 로그인되어 바로 메뉴 화면이 표시되기를
**So that** 별도 로그인 절차 없이 즉시 주문을 시작할 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 관리자가 초기 설정한 매장 식별자, 테이블 번호, 비밀번호가 로컬에 저장된다
- [ ] 저장된 정보로 자동 로그인이 수행된다
- [ ] 자동 로그인 성공 시 메뉴 화면이 표시된다
- [ ] 자동 로그인 실패 시 초기 설정 화면이 표시된다
- [ ] 페이지 새로고침 시에도 로그인 상태가 유지된다

### US-102: 관리자 로그인
**As a** 매장 오너/일반 관리자 (P-A01, P-A02)
**I want to** 매장 식별자, 사용자명, 비밀번호로 로그인하기를
**So that** 매장 관리 시스템에 안전하게 접근할 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 매장 식별자 + 사용자명 + 비밀번호 입력 폼이 제공된다
- [ ] 로그인 성공 시 JWT 토큰이 발급되고 16시간 동안 유지된다
- [ ] 브라우저 새로고침 시 세션이 유지된다
- [ ] 16시간 후 자동 로그아웃된다
- [ ] 잘못된 자격 증명 시 에러 메시지가 표시된다
- [ ] 로그인 시도 횟수가 제한된다 (연속 실패 시 일시 차단)

### US-103: 관리자 계정 생성
**As a** 매장 오너 (P-A01)
**I want to** 새로운 관리자 계정을 생성하기를
**So that** 직원에게 관리 시스템 접근 권한을 부여할 수 있다

**우선순위**: 중간

**Acceptance Criteria**:
- [ ] 매장 오너만 관리자 계정 생성 기능에 접근할 수 있다
- [ ] 사용자명, 비밀번호, 역할(오너/일반 관리자)을 입력할 수 있다
- [ ] 비밀번호는 bcrypt로 해싱되어 저장된다
- [ ] 생성 성공/실패 피드백이 표시된다

## Feature 2: 메뉴 조회 및 탐색

### US-201: 카테고리별 메뉴 조회
**As a** 고객 (P-C01, P-C02, P-C03)
**I want to** 카테고리별로 분류된 메뉴 목록을 보기를
**So that** 원하는 종류의 음식을 쉽게 찾을 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 메뉴 화면이 기본(홈) 화면으로 표시된다
- [ ] 카테고리 탭/버튼으로 메뉴가 분류되어 표시된다
- [ ] 카테고리 간 빠른 이동이 가능하다
- [ ] 각 메뉴는 카드 형태로 메뉴명, 가격, 이미지가 표시된다
- [ ] 터치 친화적 버튼 크기 (최소 44x44px)가 적용된다

### US-202: 메뉴 상세 정보 확인
**As a** 고객 (P-C01, P-C02, P-C03)
**I want to** 메뉴의 상세 정보(설명, 이미지)를 확인하기를
**So that** 주문 전에 메뉴에 대해 충분히 알 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 메뉴 카드 클릭/탭 시 상세 정보가 표시된다
- [ ] 상세 정보에 메뉴명, 가격, 설명, 이미지가 포함된다
- [ ] 상세 화면에서 장바구니에 바로 추가할 수 있다
- [ ] 닫기/뒤로가기로 메뉴 목록으로 돌아갈 수 있다

## Feature 3: 장바구니 관리

### US-301: 메뉴를 장바구니에 추가
**As a** 고객 (P-C01, P-C02, P-C03)
**I want to** 원하는 메뉴를 장바구니에 추가하기를
**So that** 주문할 메뉴를 모아둘 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 메뉴 카드 또는 상세 화면에서 장바구니 추가 버튼이 제공된다
- [ ] 추가 시 장바구니 아이콘에 수량이 표시된다
- [ ] 같은 메뉴를 다시 추가하면 수량이 증가한다
- [ ] 추가 성공 시 시각적 피드백이 제공된다

### US-302: 장바구니 수량 조절 및 삭제
**As a** 고객 (P-C01, P-C02, P-C03)
**I want to** 장바구니에서 메뉴 수량을 조절하거나 삭제하기를
**So that** 주문 전에 원하는 대로 수정할 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 장바구니에서 각 메뉴의 수량 증가/감소 버튼이 제공된다
- [ ] 수량이 0이 되면 해당 메뉴가 장바구니에서 삭제된다
- [ ] 개별 메뉴 삭제 버튼이 제공된다
- [ ] 장바구니 전체 비우기 기능이 제공된다
- [ ] 총 금액이 실시간으로 재계산된다

### US-303: 장바구니 로컬 저장
**As a** 고객 (P-C01, P-C02, P-C03)
**I want to** 페이지를 새로고침해도 장바구니 내용이 유지되기를
**So that** 실수로 페이지를 닫아도 다시 담을 필요가 없다

**우선순위**: 중간

**Acceptance Criteria**:
- [ ] 장바구니 데이터가 클라이언트 로컬 스토리지에 저장된다
- [ ] 페이지 새로고침 시 장바구니 내용이 복원된다
- [ ] 주문 확정 시 로컬 스토리지의 장바구니가 비워진다

## Feature 4: 주문 관리 (고객)

### US-401: 주문 생성
**As a** 고객 (P-C01, P-C02, P-C03)
**I want to** 장바구니의 메뉴를 주문으로 확정하기를
**So that** 매장에서 음식을 준비받을 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 주문 확정 전 주문 내역(메뉴, 수량, 금액) 최종 확인 화면이 표시된다
- [ ] 주문 확정 버튼 클릭 시 서버에 주문이 전송된다
- [ ] 주문 성공 시 주문 번호가 표시된다
- [ ] 주문 성공 시 장바구니가 자동으로 비워진다
- [ ] 주문 성공 후 메뉴 화면으로 자동 리다이렉트된다
- [ ] 주문 실패 시 에러 메시지가 표시되고 장바구니가 유지된다
- [ ] 주문 정보에 매장 ID, 테이블 ID, 메뉴 목록, 총 금액, 세션 ID가 포함된다

### US-402: 주문 내역 조회
**As a** 고객 (P-C01, P-C02, P-C03)
**I want to** 현재 테이블의 주문 내역을 확인하기를
**So that** 이전에 주문한 내용과 상태를 파악할 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 현재 테이블 세션의 주문만 표시된다 (이전 세션 제외)
- [ ] 주문이 시간 순으로 정렬된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴/수량, 금액, 상태가 표시된다
- [ ] 주문 상태는 대기중/준비중/완료로 구분된다
- [ ] 이용 완료 처리된 주문은 표시되지 않는다

### US-403: 고객 주문 상태 실시간 업데이트
**As a** 고객 (P-C01, P-C02, P-C03)
**I want to** 주문 상태가 변경되면 실시간으로 화면에 반영되기를
**So that** 음식 준비 상황을 즉시 알 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] SSE를 통해 주문 상태 변경이 실시간으로 수신된다
- [ ] 상태 변경 시 주문 내역 화면이 자동 업데이트된다
- [ ] 연결이 끊어지면 자동 재연결을 시도한다

## Feature 5: 실시간 주문 모니터링 (관리자)

### US-501: 실시간 주문 대시보드
**As a** 일반 관리자 (P-A02)
**I want to** 들어오는 주문을 실시간 대시보드에서 확인하기를
**So that** 주문을 빠르게 접수하고 처리할 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 테이블별 카드 형태의 그리드 레이아웃이 표시된다
- [ ] 각 테이블 카드에 총 주문액과 최신 주문 미리보기가 표시된다
- [ ] SSE를 통해 신규 주문이 2초 이내에 화면에 표시된다
- [ ] 신규 주문 시 시각적 강조(색상 변경, 애니메이션)가 적용된다
- [ ] 테이블별 필터링이 가능하다

### US-502: 주문 상세 보기
**As a** 일반 관리자 (P-A02)
**I want to** 테이블 카드를 클릭하여 전체 주문 상세를 보기를
**So that** 해당 테이블의 모든 주문 내용을 파악할 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 테이블 카드 클릭 시 해당 테이블의 전체 주문 목록이 표시된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴/수량, 금액, 상태가 표시된다
- [ ] 닫기 버튼으로 대시보드로 복귀할 수 있다

### US-503: 주문 상태 변경
**As a** 일반 관리자 (P-A02)
**I want to** 주문 상태를 변경하기를 (대기중 → 준비중 → 완료)
**So that** 주문 처리 진행 상황을 관리할 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 각 주문에 상태 변경 버튼이 제공된다
- [ ] 상태는 대기중 → 준비중 → 완료 순서로 변경된다
- [ ] 상태 변경 시 고객 화면에도 실시간 반영된다 (SSE)
- [ ] 상태 변경 성공/실패 피드백이 표시된다

## Feature 6: 테이블 관리

### US-601: 테이블 초기 설정
**As a** 매장 오너 (P-A01)
**I want to** 테이블 번호와 비밀번호를 설정하기를
**So that** 고객 태블릿이 해당 테이블로 자동 로그인할 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 테이블 번호와 비밀번호를 입력할 수 있다
- [ ] 설정 완료 시 16시간 세션이 생성된다
- [ ] 설정 정보가 저장되고 자동 로그인이 활성화된다
- [ ] 설정 성공/실패 피드백이 표시된다

### US-602: 주문 삭제 (직권 수정)
**As a** 일반 관리자 (P-A02)
**I want to** 특정 주문을 삭제하기를
**So that** 잘못된 주문이나 취소 요청을 처리할 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 각 주문에 삭제 버튼이 제공된다
- [ ] 삭제 전 확인 팝업이 표시된다
- [ ] 삭제 시 주문이 즉시 제거된다
- [ ] 테이블 총 주문액이 재계산된다
- [ ] 삭제 성공/실패 피드백이 표시된다

### US-603: 테이블 이용 완료 처리
**As a** 일반 관리자 (P-A02)
**I want to** 테이블 이용 완료를 처리하기를
**So that** 해당 세션의 주문을 마감하고 새 고객을 받을 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 테이블 카드에 이용 완료 버튼이 제공된다
- [ ] 이용 완료 전 확인 팝업이 표시된다
- [ ] 이용 완료 시 해당 세션의 주문 내역이 과거 이력으로 이동된다
- [ ] 테이블의 현재 주문 목록과 총 주문액이 0으로 리셋된다
- [ ] 새 고객이 이전 주문 내역 없이 시작할 수 있다
- [ ] 성공/실패 피드백이 표시된다

### US-604: 과거 주문 내역 조회
**As a** 매장 오너 (P-A01)
**I want to** 테이블별 과거 주문 내역을 조회하기를
**So that** 이전 매출과 주문 이력을 확인할 수 있다

**우선순위**: 중간

**Acceptance Criteria**:
- [ ] 과거 내역 버튼 클릭 시 과거 주문 목록이 표시된다
- [ ] 테이블별로 과거 주문이 시간 역순으로 정렬된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
- [ ] 날짜 필터링 기능이 제공된다
- [ ] 닫기 버튼으로 대시보드로 복귀할 수 있다

## Feature 7: 메뉴 관리

### US-701: 메뉴 등록
**As a** 매장 오너 (P-A01)
**I want to** 새로운 메뉴를 등록하기를
**So that** 고객에게 새로운 메뉴를 제공할 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 메뉴명, 가격, 설명, 카테고리, 이미지를 입력할 수 있다
- [ ] 이미지는 파일 업로드로 등록할 수 있다
- [ ] 필수 필드(메뉴명, 가격, 카테고리) 검증이 수행된다
- [ ] 가격 범위 검증이 수행된다 (0 이상)
- [ ] 등록 성공/실패 피드백이 표시된다

### US-702: 메뉴 수정
**As a** 매장 오너 (P-A01)
**I want to** 기존 메뉴 정보를 수정하기를
**So that** 가격 변경이나 설명 업데이트를 반영할 수 있다

**우선순위**: 높음

**Acceptance Criteria**:
- [ ] 메뉴 목록에서 수정 버튼이 제공된다
- [ ] 기존 정보가 폼에 미리 채워진다
- [ ] 이미지를 새로 업로드하여 교체할 수 있다
- [ ] 수정 시 동일한 필드 검증이 수행된다
- [ ] 수정 성공/실패 피드백이 표시된다

### US-703: 메뉴 삭제
**As a** 매장 오너 (P-A01)
**I want to** 메뉴를 삭제하기를
**So that** 더 이상 제공하지 않는 메뉴를 제거할 수 있다

**우선순위**: 중간

**Acceptance Criteria**:
- [ ] 메뉴 목록에서 삭제 버튼이 제공된다
- [ ] 삭제 전 확인 팝업이 표시된다
- [ ] 삭제 성공/실패 피드백이 표시된다

### US-704: 메뉴 조회 (관리자)
**As a** 매장 오너 (P-A01)
**I want to** 카테고리별로 등록된 메뉴를 조회하기를
**So that** 현재 메뉴 구성을 파악하고 관리할 수 있다
