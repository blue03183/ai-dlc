# 테이블오더 서비스 — 페르소나 정의

---

## 고객 페르소나 (이용 패턴별)

### P-C01: 혼밥 고객 (Solo Diner)
- **이름**: 김민수 (30대 직장인)
- **특징**: 혼자 식사, 빠른 주문과 식사를 선호
- **목표**: 최소한의 인터랙션으로 빠르게 주문 완료
- **동기**: 점심시간이 짧아 효율적인 주문이 중요
- **불편사항**: 복잡한 UI, 불필요한 단계가 많은 주문 과정
- **이용 시나리오**: 메뉴를 빠르게 훑어보고, 익숙한 메뉴를 바로 장바구니에 담아 주문

### P-C02: 단체 고객 (Group Diner)
- **이름**: 이지영 (20대 대학생 그룹 대표)
- **특징**: 3~6명 단체 식사, 여러 메뉴를 함께 고르고 추가 주문이 잦음
- **목표**: 여러 메뉴를 편리하게 담고, 추가 주문을 쉽게 하기
- **동기**: 모임에서 다양한 메뉴를 시도하고 싶음
- **불편사항**: 장바구니에서 수량 조절이 불편한 경우, 추가 주문 시 처음부터 다시 시작해야 하는 경우
- **이용 시나리오**: 메뉴를 천천히 탐색하며 여러 개를 장바구니에 담고, 식사 중 추가 주문을 반복

### P-C03: 상습 방문 고객 (Regular Customer)
- **이름**: 박철호 (40대 자영업자)
- **특징**: 같은 매장을 주 2~3회 방문, 단골 메뉴가 있음
- **목표**: 익숙한 메뉴를 빠르게 찾아 주문
- **동기**: 매번 같은 메뉴를 주문하므로 빠른 접근이 중요
- **불편사항**: 매번 메뉴를 처음부터 탐색해야 하는 경우
- **이용 시나리오**: 메뉴 화면에서 익숙한 카테고리로 바로 이동, 단골 메뉴를 빠르게 선택

## 관리자 페르소나 (2단계)

### P-A01: 매장 오너 (Store Owner)
- **이름**: 정수진 (50대 식당 사장)
- **특징**: 매장 전체 운영 책임, 메뉴/가격 결정권, 직원 관리
- **목표**: 매장 운영 현황을 한눈에 파악하고, 메뉴와 가격을 관리
- **동기**: 매출 극대화, 운영 효율성 향상
- **불편사항**: 복잡한 관리 화면, 실시간 상황 파악이 어려운 경우
- **주요 기능**: 메뉴 관리(등록/수정/삭제), 관리자 계정 생성, 테이블 초기 설정, 과거 주문 내역 조회
- **권한**: 모든 관리 기능 접근 가능

### P-A02: 일반 관리자 (Staff Manager)
- **이름**: 한동우 (20대 매장 매니저)
- **특징**: 현장 운영 담당, 주문 접수 및 상태 관리
- **목표**: 들어오는 주문을 빠르게 확인하고 처리
- **동기**: 고객 대기 시간 최소화, 원활한 매장 운영
- **불편사항**: 주문이 밀릴 때 실시간 파악이 어려운 경우
- **주요 기능**: 실시간 주문 모니터링, 주문 상태 변경, 테이블 이용 완료 처리, 주문 삭제
- **권한**: 주문/테이블 운영 기능 접근 가능 (메뉴 관리, 계정 생성 제외 가능)
